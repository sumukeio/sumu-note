# SegmentedEditor 测试指南

## 测试页面

访问测试页面：`/test-segmented-editor`

## 测试步骤

### 1. 基础功能测试

#### 测试表格编辑
- [ ] 点击表格单元格，检查是否可以正常编辑
- [ ] 输入内容后，检查 Markdown 源码是否正确更新
- [ ] 检查光标位置是否正常（不应该瞬移）

#### 测试表格操作
- [ ] 点击"添加行"按钮，检查是否成功添加新行
- [ ] 点击"添加列"按钮，检查是否成功添加新列
- [ ] 悬停表格，点击删除按钮，检查是否成功删除行/列
- [ ] 检查删除后表格结构是否正常

#### 测试文本编辑
- [ ] 在文本段中输入内容，检查是否正常
- [ ] 检查光标位置是否正常
- [ ] 检查 Markdown 源码是否正确同步

### 2. 边界情况测试

#### 空内容
- [ ] 清空所有内容，检查是否正常显示
- [ ] 输入新内容，检查是否正常解析

#### 多个表格
- [ ] 创建包含多个表格的内容
- [ ] 检查每个表格是否独立显示和编辑
- [ ] 检查表格之间的文本段是否正常

#### 表格在开头/结尾
- [ ] 创建表格在内容开头的场景
- [ ] 创建表格在内容结尾的场景
- [ ] 检查解析和显示是否正常

### 3. 性能测试

- [ ] 创建包含 10+ 个表格的内容
- [ ] 检查页面渲染是否流畅
- [ ] 检查编辑响应是否及时
- [ ] 检查是否有明显的性能问题

### 4. 错误检查

- [ ] 打开浏览器控制台
- [ ] 检查是否有 React 错误
- [ ] 检查是否有控制台警告
- [ ] 检查是否有 TypeScript 类型错误

### 5. 兼容性测试

- [ ] 测试 Chrome/Edge
- [ ] 测试 Firefox
- [ ] 测试 Safari（如果可用）
- [ ] 测试移动端浏览器（如果可用）

## 已知问题

### 问题1：文本段之间的空行处理
**描述**：段之间使用 `\n\n` 分隔，可能导致空行显示不一致
**状态**：待优化

### 问题2：表格解析可能不完整
**描述**：复杂的表格格式可能解析失败
**状态**：待测试
