# 移动端测试指南

## 📱 在手机上测试本地开发服务器

### 步骤 1: 启动移动端开发服务器

在电脑上运行：

```bash
npm run dev:mobile
```

这会启动开发服务器并允许局域网访问。

### 步骤 2: 查看你的电脑 IP 地址

**Windows:**
```powershell
ipconfig
```
找到 "IPv4 地址"，例如：`10.10.10.126`

**Mac/Linux:**
```bash
ifconfig | grep "inet "
```
找到局域网 IP，例如：`192.168.1.100`

### 步骤 3: 确保手机和电脑在同一 Wi-Fi 网络

确保你的手机和电脑连接到**同一个 Wi-Fi 网络**。

### 步骤 4: 在手机上访问

在手机浏览器中打开：

```
http://你的IP地址:3000
```

例如：`http://10.10.10.126:3000`

### 步骤 5: 测试内容

✅ 检查编辑页内容区域是否覆盖全屏
✅ 检查按钮栏是否完整显示（可以横向滚动）
✅ 测试所有交互功能
✅ 测试横竖屏切换
✅ 测试触摸手势（滑动返回等）

## 🔧 浏览器开发者工具（桌面端模拟）

### Chrome/Edge

1. 按 `F12` 打开开发者工具
2. 点击设备工具栏图标，或按快捷键：
   - Windows: `Ctrl + Shift + M`
   - Mac: `Cmd + Option + M`
3. 选择设备：
   - iPhone 14 Pro (393×852)
   - iPhone SE (375×667)
   - Samsung Galaxy S21 (360×800)
   - iPad (768×1024)

### Firefox

1. 按 `F12` 打开开发者工具
2. 点击"响应式设计模式"图标
3. 或按快捷键：`Ctrl + Shift + M` (Mac: `Cmd + Option + M`)

### Safari (Mac only)

1. 启用开发者菜单：
   - Safari → 偏好设置 → 高级 → 勾选"在菜单栏中显示开发菜单"
2. 开发 → 进入响应式设计模式

## 🐛 常见问题

### 问题 1: 手机无法访问 `http://IP:3000`

**解决方法：**
1. 确保电脑防火墙允许端口 3000
   - Windows: 控制面板 → Windows Defender 防火墙 → 允许应用通过防火墙 → 允许 Node.js
   - Mac: 系统设置 → 网络 → 防火墙 → 选项 → 允许传入连接
2. 确保手机和电脑在同一 Wi-Fi 网络
3. 确保使用 `npm run dev:mobile` 而不是 `npm run dev`

### 问题 2: 样式在手机上显示不正确

**可能原因：**
- 使用了 `100vh` 而不是 `100dvh`（已修复）
- Tailwind 的响应式断点未正确设置
- 移动端浏览器有默认样式覆盖

### 问题 3: 触摸事件不响应

**检查：**
- 确保使用了正确的事件处理器（如 `onTouchStart` 而非仅 `onClick`）
- 检查 CSS 中是否有 `pointer-events: none` 等样式
