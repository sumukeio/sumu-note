## 移动端表格显示优化建议

结合你提到的这些产品在移动端的做法，可以这样优化：

---

### 一、常见做法概览

| 产品 | 表格处理方式 |
|------|--------------|
| **iOS / 三星 / 小米 备忘录** | 表格较少，多为简单列表；有表格时多支持横向滑动查看 |
| **飞书 / 语雀** | 表格可横向滚动，表头/首列可固定；部分支持卡片视图 |
| **印象笔记 / 有道云** | 以横向滚动为主，辅以字体和行高控制 |
| **石墨 / Notion** | Notion 的表格可横向滚动；部分产品提供“简化视图”（卡片化） |
| **浮墨** | 更偏向短文本，表格使用较少 |

---

### 二、推荐优化方向（按优先级）

#### 1. 横向滚动 + 视觉提示（优先）

- 表格容器设 `overflow-x: auto`，支持左右滑动。
- 使用渐变/阴影暗示“右侧还有内容”，例如：右侧 `linear-gradient(to right, transparent, rgba(0,0,0,0.05))` 或细阴影。
- 给表格容器加 `-webkit-overflow-scrolling: touch` 提升滑动流畅度。
- 表格设置 `min-width`，避免列被压缩得太窄。

#### 2. 固定表头（可选）

- 表头 `position: sticky; top: 0; z-index: 1; background: ...`。
- 保证表头背景不透明，避免滚动时内容穿透。

#### 3. 字体与行距（中）

- 表格内字体略小于正文（如 `text-sm`），控制单元格 `padding`。
- 适当增大行高，但避免过度导致行数过多。

#### 4. 卡片化 / 简化视图（进阶）

- 将“多列表格”在移动端转为“每行一条”的卡片展示。
- 每张卡片：字段名 + 值，垂直堆叠，适合列数较多的表格。
- 可作为模式切换（表格 ↔ 卡片），或在窄屏下自动使用卡片视图。

#### 5. 缩放与布局

- 表格不跟随整页缩放，保持固定宽度（配合横向滚动）。
- 或在首屏展示“缩略视图”，点击进入全表/全屏表格模式。
- 分页表格可考虑减少每页行数，降低单屏信息量。

---

### 三、实现成本与取舍

| 方案 | 实现难度 | 效果 |
|------|----------|------|
| 横向滚动 + 渐变提示 | 低 | 高，能立即改善体验 |
| 固定表头 | 低 | 中 |
| 字体/行距微调 | 低 | 中 |
| 卡片化视图 | 高 | 高（尤其对多列表格） |
| 全屏表格模式 | 中 | 中 |

---

### 四、最小改动、收益最大的组合

1. 表格容器：`overflow-x: auto` + `min-width` 保证可横向滑动且列宽不被压扁。  
2. 右侧渐变或 shadow：提示“可向右滑动”。  
3. 移动端表格：`text-sm`，适当减小单元格 `padding`，保证文字不被裁剪。  
4. 表头：`sticky` 固定，滚动时始终可见。

## 实现状态（已落地）

已按「最小改动、收益最大」组合完成实现：

### MarkdownRenderer（预览 / 阅读态）
- 表格容器：`overflow-x-auto` + `WebkitOverflowScrolling: touch`
- 表格：`min-w-max` 避免列被压扁
- 右侧渐变：`bg-gradient-to-l from-background to-transparent`，仅移动端（`sm:hidden`）
- 表头：`sticky top-0 z-10 bg-muted/95`
- 单元格：移动端 `px-2 py-1.5 text-xs`，桌面端 `px-4 py-2 text-sm`；表头 `whitespace-nowrap`

### SegmentedEditor（编辑态）
- 同样应用横向滚动、渐变提示、表头 sticky、响应式 padding 与字号
- 渐变使用 `from-card` 以匹配卡片背景
