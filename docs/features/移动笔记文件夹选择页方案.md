# 移动笔记/文件夹 - 目标文件夹选择页方案

**需求**：移动笔记或文件夹到其他文件夹时，文件夹选择页面参考飞书 / 语雀 / Notion / iOS 备忘录 / 小米笔记 / 三星笔记 / 石墨 / 印象笔记 / 有道云，给出统一方案。

**当前实现**（`NoteManager` 内「移动到...」弹窗）：
- 标题「移动到...」+ 说明文案
- 「上次移动到」快捷入口（带虚线框）
- 目标列表：**全量展开的树**，用 `paddingLeft` 做缩进，无折叠、无搜索、无根目录入口
- 目标仅限「其他文件夹」，不支持移动到**根目录**（`folder_id = null`）
- 移动文件夹时仅排除「被选中的文件夹本身」，未排除其**子级/后代**，存在循环引用风险

---

## 一、参考产品共性（简要）

| 产品 | 选择方式 | 根/全部 | 搜索 | 树形 | 其他 |
|------|----------|--------|------|------|------|
| 飞书 | 树 + 面包屑/路径 | 有「全部」或根 | 支持按名称筛选 | 可折叠 | 最近使用/上次位置 |
| 语雀 | 知识库 + 树 | 有顶层 | 支持 | 层级清晰 | 路径展示 |
| Notion | 页面树 / 侧边栏 | 有「顶层」 | 支持 | 可折叠 | 块/页面一致 |
| iOS 备忘录 | 列表 | 「On My iPhone」等 | 无 | 缩进列表 | 极简 |
| 小米/三星笔记 | 列表 | 根/默认文件夹 | 部分有 | 缩进 | 偏系统风格 |
| 石墨 | 树 + 弹窗 | 有根 | 支持 | 可折叠 | 与侧边栏一致 |
| 印象笔记 | 笔记本/笔记本组 | 有「默认」 | 支持 | 层级 | 快捷最近 |
| 有道云 | 文件夹树 | 有根 | 支持 | 可折叠 | 路径可选 |

**共性结论**：
1. **必有「根/全部」**：支持移动到「未分类/根目录」。
2. **树形展示**：层级用缩进或折叠树，与主界面侧边栏/列表一致。
3. **可折叠**：层级多时支持折叠父节点，减少滚动。
4. **搜索**：文件夹多时按名称快速定位（飞书/语雀/Notion/石墨/印象/有道）。
5. **快捷**：上次移动目标、最近使用等（我们已有「上次移动到」）。
6. **防循环**：移动文件夹时，不能选自己及其子/孙文件夹。

---

## 二、推荐方案总览

- **根目录**：增加「根目录 / 全部笔记」选项，对应 `folder_id = null`。
- **树形 + 折叠**：保留层级树，支持按节点折叠/展开，默认可「全部展开」或「只展开到第 2 层」。
- **搜索**：顶部增加搜索框，按文件夹名称过滤，高亮匹配；空关键词显示完整树。
- **快捷**：保留「上次移动到」；可选增加「当前所在文件夹」快捷（仅移动笔记时）。
- **禁用规则**：移动文件夹时，从候选中排除「被移动文件夹及其所有后代」，避免循环引用。
- **移动端**：同一逻辑，弹窗内可缩小字体、加大点击区域、支持触控折叠。

## 三、交互与 UI 细则

### 3.1 弹窗结构（自上而下）

```
┌─────────────────────────────────────────┐
│  移动到...                           [X] │
│  选择目标文件夹（可选根目录）             │
├─────────────────────────────────────────┤
│  [🔍 搜索文件夹名称]                      │
├─────────────────────────────────────────┤
│  ┌─ 上次移动到 ─────────────────────┐   │
│  │  📁 某文件夹名                    │   │  ← 保留现有
│  └──────────────────────────────────┘   │
│  ┌─ 选择目标 ───────────────────────┐   │
│  │  📁 根目录 / 全部笔记             │   │  ← 新增，对应 null
│  │  ▶ 📁 工作                       │   │  ← 可折叠，▶ 展开为 ▼
│  │      📁 项目A                    │   │
│  │      📁 项目B                    │   │
│  │  ▼ 📁 生活                       │   │
│  │      📁 读书                     │   │
│  │      ...                         │   │
│  └──────────────────────────────────┘   │
│  （列表区域可滚动，max-height 约 50vh）   │
└─────────────────────────────────────────┘
```

### 3.2 根目录

- **文案**：统一用「根目录」或「全部笔记」（与产品用语一致），占一行，放在树的最上方。
- **行为**：点击后目标为 `null`；笔记的 `folder_id` 更新为 `null`，文件夹的 `parent_id` 更新为 `null`。
- **后端**：笔记表已支持 `folder_id` 为 null；文件夹表需支持 `parent_id` 为 null（根级文件夹），当前已支持。

### 3.3 树形与折叠

- **数据**：继续用现有 `FolderItem[]` + `parent_id` 建树；先按 `parent_id` 分组，再按名称排序，递归生成树。
- **展示**：每一行为「折叠图标 + 图标 + 名称」；点击行选为目标，点击折叠图标仅展开/收起子节点。
- **默认**：建议默认全部展开；若文件夹很多（如 >50），可改为默认只展开前两层，其余折叠。
- **缩进**：与现有一致，用 `paddingLeft: 12 + depth * 16` 或使用统一变量（如 16px/级）。

### 3.4 搜索

- **位置**：弹窗顶部、说明文案下方。
- **行为**：输入时对文件夹名称做**模糊匹配**（不区分大小写），只展示匹配的节点；若某节点匹配，则展示其**从根到该节点的路径**上的所有祖先，并展开到该节点（子节点可折叠）。
- **空结果**：无匹配时展示「未找到匹配的文件夹」。
- **清空**：清空搜索框即恢复完整树。

### 3.5 禁用规则（防止循环）

- **移动笔记**：所有文件夹 + 根目录 均可选，无需排除。
- **移动文件夹**：从候选中排除：
  - 被选中的文件夹本身（已实现）；
  - 被选中文件夹的**所有后代**（子、孙、…）。
- **实现**：在拉取全部文件夹后，对每个被移动的 folderId 做一次「收集其所有后代 id」；最终 `targets = 全部文件夹.filter(f => !被移动id集合.has(f.id) && !后代id集合.has(f.id))`；根目录始终可选。

### 3.6 上次移动到 / 当前文件夹

- **上次移动到**：保持现有逻辑与展示；若上次目标是「根目录」，则快捷按钮文案为「根目录」或「全部笔记」。
- **当前文件夹**（可选）：若只移动笔记、且当前在某个文件夹内，可增加「移动到当前文件夹」快捷（即 `currentFolderId`），减少误移出后的重复操作。

## 四、数据与接口

### 4.1 根目录

- **笔记**：`moveNotesToFolder(noteIds, userId, null)` 与 `updateNote(..., { folder_id: null })` 需支持 `targetFolderId = null`；若 `note-service` 当前仅接受 string，可改为 `string | null` 并在内部用 `.is('folder_id', null)` 更新。
- **文件夹**：移动文件夹到根时，`parent_id` 更新为 `null`；Supabase 已支持。

### 4.2 获取「全部文件夹」与排除后代

- 继续用 `supabase.from('folders').select('*').eq('user_id', userId)` 拉全量。
- 在前端实现 `getDescendantIds(folders, folderId): Set<string>`（按 parent_id 建图，从 folderId 起 BFS/DFS 收集所有后代 id）。
- `allowedTargets = folders.filter(f => !movedIds.has(f.id) && !descendantIds.has(f.id))`；根目录单独加一项，不参与过滤。

### 4.3 组件与状态建议

- **状态**：`targetList: FolderItem[]`（已过滤后的目标文件夹）、`expandedIds: Set<string>`（展开的文件夹 id）、`searchQuery: string`、`lastMoveTargetId: string | null`（可改为 `string | null`，`null` 表示上次是根目录）。
- **组件**：可抽成 `MoveToFolderDialog`，接收 `open`、`onOpenChange`、`moveTargets`（要移动的文件夹/笔记信息）、`onConfirm(targetFolderId: string | null)`、`lastMoveTargetId`、`currentFolderId?: string` 等，便于在 NoteManager 与后续 FolderManager 复用。

## 五、与 FolderManager 的移动弹窗统一

- **FolderManager** 当前在根级只展示「根文件夹」列表做移动目标，且无子文件夹、无根目录选项；若产品希望「在根级移动文件夹时也能选到子文件夹或根目录」，建议：
  - 拉取**全部文件夹**（与 NoteManager 一致），
  - 应用同一套「树形 + 折叠 + 搜索 + 根目录 + 排除自身与后代」规则，
  - 复用同一 `MoveToFolderDialog` 组件，仅传入「只移动文件夹」的 payload 与文案（如「选择目标文件夹」）。

## 六、实现优先级建议

| 优先级 | 项 | 说明 |
|--------|----|------|
| P0 | 根目录选项 | 产品标配，且数据已支持 |
| P0 | 排除后代，防循环 | 移动文件夹时的数据正确性 |
| P1 | 树形折叠 | 层级多时体验明显提升 |
| P1 | 搜索 | 文件夹多时必备 |
| P2 | 抽 MoveToFolderDialog 复用 | 与 FolderManager 统一、易维护 |
| P2 | 当前文件夹快捷 | 按产品需求取舍 |

## 七、小结

- **根目录**：必选，对应 `folder_id` / `parent_id` 为 null。
- **树形 + 折叠**：与飞书/语雀/Notion/石墨等一致，默认可全展开。
- **搜索**：按名称过滤并展示路径，无匹配时明确提示。
- **快捷**：保留「上次移动到」，并支持上次选根目录。
- **安全**：移动文件夹时排除自身及所有后代，防止循环引用。
- **统一**：NoteManager 与 FolderManager 共用一个「移动到…」弹窗逻辑与组件，便于后续扩展（如多选目标、批量移动到同一文件夹等）。

以上方案可直接作为开发规格使用；若需要，我可以再按「P0 → P1」拆成具体任务列表或补全 `MoveToFolderDialog` 的 props/接口定义。
